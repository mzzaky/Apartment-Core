Version 1.2.9: Auction Feature
Release Date: September 17, 2025
============================================================

--- Added - Auction System ---

* Complete auction subsystem for apartments
    - Model: `ApartmentAuction` stores apartment-id, owner, bids, times, and flags.
    - Manager: `AuctionManager` handles all auction logic.
    - Persistence: Active auctions saved to the plugin data file.
    - Concurrency: Managed in a `ConcurrentHashMap` for safe access.

* Player commands
    - `/apartmentcore auction create <id> <startingBid> <durationHours>`
    - `/apartmentcore auction bid <id> <amount>`
    - `/apartmentcore auction list [<filter>]`
    - `/apartmentcore auction cancel <id>`

--- Admin - Auction Management ---

* New admin auction subcommands:
    - `/apartmentcore admin auction list [filter]`
    - `/apartmentcore admin auction cancel <id>`
    - `/apartmentcore admin auction forceend <id>`

--- Configuration & Rules ---

* Config-driven options for cooldown, fee, bid limits, duration, commission, and broadcasts.
* Rules enforced: Owner must own the apartment, cannot auction with unpaid taxes, cannot sell while in auction.

--- Payments, Refunds & Ownership Transfer ---

* Vault economy integration for all transactions.
* On successful auction end, winner becomes owner and seller is paid (minus commission).

--- Persistence & Safety ---

* Auctions stored under `auctions` in the plugin data file.
* Serialization handled by `ApartmentAuction.serialize()` and `deserialize()`.

--- Messages & Localization ---

* Rebuilt the messages system to be minimal, focused and reloadable in-game via `/apartmentcore admin reload`.
* `messages.yml` now contains only essential text.
* Fallback behavior for missing message keys.
* `{prefix}` and color code support.

Version 1.2.8: Major Update
Release Date: September 13, 2025
============================================================

--- Critical Bug Fixes ---

* Thread-Safety Improvements: Fixed critical thread-safety issue in auto-save system by moving task to main thread.
* UUID Parsing & Validation: Enhanced `loadApartments()` to validate owner string before UUID parsing.
* Null Pointer Exception Prevention: Added additional null checks in teleport logic and economy usage.

--- Architecture Improvements ---

* Service Layer Extraction: Created `ApartmentCommandService.java` and extracted command logic from `CommandHandler`.
* Class Structure Reorganization: Moved inner classes `ApartmentRating` and `GuestBookEntry` to standalone files.

--- Configuration & Resource Management ---

* Messages Configuration System: Added a complete `messages.yml` file with extensive options.
* Plugin Manifest Fixes: Fixed `plugin.yml` schema validation errors.
* Statistics File Population: Fixed empty `apartments-stats.yml` on fresh installations.

--- User Experience Enhancements ---

* Smart Tab Completion: Enhanced TabCompleter logic for more intuitive command suggestions.
* Simplified Admin Commands: Simplified `/apartmentcore admin create` syntax from 5 to 3 arguments.

--- Admin Features ---

* Backup System Implementation: Added a complete backup command system (`create`, `list`, `restore`).
* Logging System: Implemented comprehensive logging via `LoggerManager.java` with file rotation and size limits.

--- Code Quality Improvements ---

* Efficient Resource Usage and improved error handling.
