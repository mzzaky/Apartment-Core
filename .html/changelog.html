Version 1.3.8

Edition System
March 20, 2026
▼ Minimize
Release Overview
Changes
30+
New Feature
Edition System (Free & Pro)
Type
Massive Update
+
Features & Improvements
  • Added license and plugin edition information on /ac version.
  • Added license status and edition check report on /ac admin reload (English).
  • Introduced dual-edition build system using Maven profiles: Free (default) and Pro.
  • Added license validation system for Pro edition using Cloudflare Workers + Supabase infrastructure.
  • Async validation on startup, 7-day offline grace period, and SHA-256 hashed local cache for Pro edition.
  • Added Cloudflare Worker API code (infrastructure/cloudflare-worker/) and Supabase database schema for license management.
  • Added license.key configuration field in config.yml for Pro edition.
  • Centralized all shop messages into messages.yml and removed the redundant messages section from shop.yml.
  • Implemented shop message usage in ApartmentShopGUI and ApartmentShopManager to replace hardcoded strings.
  • Standardized placeholder format to {placeholder} across all shop-related messages.
⌘
Edition System Comparison
  • Free Edition Limits:
    • Maximum 20 apartments can be registered on the server.
    • Maximum apartment level capped at 5.
    • Research system available with hardcoded values (cost, duration, effects).
    • Configuration files (Research, Shop, Achievements, GUI) are not exported to disk.
    • Logging and backup features are disabled.
  • Pro Edition Features:
    • Unlimited apartment creation and levels.
    • Fully customizable Research, Shop, Achievement, and GUI systems via YAML files.
    • Full logging features (transaction and admin action logs).
    • Full backup features (auto-backup and manual backup commands).
↑
Changes
  • Updated splash banner (SplashArt) to display edition info and module availability.
  • Updated plugin version to 1.3.8 and config-version to 4.
  • Moved apartment levels configuration from config.yml to a new dedicated levels.yml file.
  • Core features (Income Generation, Tax System, Teleportation) are now always enabled and non-toggleable. Corresponding configuration toggles have been removed from config.yml.
  • Updated research system to be visible and functional in Free edition, although configuration remains restricted to Pro.
  • Refined config.yml and adjusted it to the latest updates.
  • Cleaned up shop.yml by removing unused sections: gui, performance, debug, compatibility, and config_version.
  • Removed unused shop system settings from shop.yml: enabled, allow_custom_costs, and refund_percentage.
  • Removed unused gui block from research.yml.
⚙
Technical & Fixes
  • Introduced Maven profiles: edition-free (default) and edition-pro.
  • New Maven properties for build automation: apartmentcore.edition, apartmentcore.edition.label, and apartmentcore.jar.classifier.
  • Build commands:
    • Free: mvn clean package (Outputs: ApartmentCore-Free-1.3.8.jar)
    • Pro: mvn clean package -Pedition-pro (Outputs: ApartmentCore-Pro-1.3.8.jar)
  • Added Edition enum, EditionManager, and LicenseManager classes under com.aithor.apartmentcore.edition.
  • Enabled Maven resource filtering for plugin.yml variable substitution (e.g., edition field).
  • EditionManager provides centralized gating for: getMaxApartments(), getMaxLevel(), isResearchCustomisable(), isLoggingEnabled(), and isBackupEnabled().
  • The "Pending Income" capacity in menus now includes Vault Expansion stats from the shop.
  • Fixed: shop.yml was created as empty (0 bytes) in Free edition because EditionManager was initialized after ConfigManager.loadConfiguration().

Balancing
  • Changed the default apartment upgrade duration to 1-5 days (updated the comment help information in the configuration).

Added
  • YAML configuration system for the Main Menu GUI via the custom_gui/main_menu.yml file.
    • GUI title (gui_title), GUI size (gui_size), and borders are now configurable.
    • Each menu item supports configuration: slot, material, custom_model_data, name, lore, and glow.

    • GUI size validation (must be a multiple of 9, between 9 and 54).
↑
Changed
  • MainMenuGUI.java now completely reads the layout from the YAML configuration file.
    • All hardcoded values have been made fallbacks, not the main source.
    • Menu item slot positions are now dynamic, following the configuration.
    • Click handlers use slot positions from the configuration.
    • Placeholders are resolved at runtime to dynamically display player data.
  • ApartmentCore.java initializes MainMenuConfig when the plugin is enabled.
ℹ
Notes

  • All existing values in the Java files still function as fallbacks if the configuration is incomplete.

+
New Features
  • Added Vault Expansion shop item to shop.yml which increases the apartment's maximum income capacity by a percentage (2.5% per tier).
    • Implemented INCOME_CAPACITY enum in ShopBuffType.java to handle capacity percentage buffs.
    • Added VAULT_EXPANSION to ShopItem.java using the ENDER_CHEST material icon.
    • Updated ApartmentShopManager and ApartmentManager to calculate and apply the new capacity buff to apartment income generation.
    • Updated ApartmentShopGUI to display the new Vault Expansion shop item and its active buffs.
    • Updated ApartmentStatisticsGUI to show the active capacity buff multiplier applied from the shop on the "Final Math" and "Other Stats" panels.
  • Achievement unlock now triggers 4 distinct effects:
    • Sound effect (configurable Bukkit sound, volume & pitch)
    • Title message (configurable header, subheader, fade timings)
    • Action bar / toast message (configurable format string)
    • Animated BossBar that drains over a set duration (configurable color, style, duration)
  • Added detailed unlock timestamp (date and time with seconds) to the Achievement GUI lore for completed achievements.
  • Added multi-reward system for achievements. Rewards now support multiple entries per achievement with 3 types:
    • [money] <amount> - Give money via economy
    • [exp] <amount> - Give experience points
    • [console] <command> - Execute a console command (%player% placeholder supported)
    • Backward compatible: old single reward: <number> config still works.
  • Configurable GUI Lore: The layout and formatting of achievement items in the GUI are now fully customizable via achievements.yml.
    • Define global defaults under achievements.format.
    • Override defaults on a per-achievement basis under definitions.<achievement>.format.
    • Includes support for completed, in-progress, and disabled states, plus dynamic placeholders like {progress_bar}, {percentage}, and {completed_date}.
↑
Changed
  • Comprehensive Apartment Statistics GUI: Completely overhauled the Apartment Statistics menu layout to provide deeper insights into the apartment economy.
    • Transformed from a functional menu with action buttons into a detailed, multi-panel statistics overview (expanded to a 54-slot inventory).
    • Base Stats Panel: Displays raw apartment attributes from the level config (min/max income, base capacity, tax rate).
    • Shop Buffs Panel: Shows all applied shop upgrades (Tiers 1-5) and their cumulative effects on income, tax, and guestbook space.
    • Research Buffs Panel: Lists active player-wide research buffs (e.g., Capital Growth, Tax Efficiency) and their current tier bonuses.
    • Final Math Panel: Provides a step-by-step mathematical breakdown of how final income per cycle, income capacity, and tax are calculated by sequentially stacking all buffs (Flat -> Shop% -> CapitalGrowth%).
    • Other Stats Panel: Summarizes lifetime earnings, total tax paid, pending income (with % of capacity), and active invoices/status.
    • Action buttons (Claim Income, Tax Mgmt, Auto-Pay toggle) have been removed from this view to centralize apartment management inside the Apartment Details menu.
  • Tax System Configurable Percentage: The required base tax amount (previously hardcoded at 2.5% of apartment price per level) is now fully configurable under the apartment-levels section in config.yml.
    • Each level now accepts a tax-percentage value (e.g., 2.5 for Level 1, 5.0 for Level 2). This percentage determines the base invoice amount before any other multipliers (like late fees, shop buffs, or research effects).
✕
Fixed
  • Fixed a bug where the tax generation timer would reset without generating a new tax bill after a server restart. The last-minecraft-day value is now properly loaded from storage.
  • Fixed a bug causing the income generation timer in the GUI to incorrectly display "Soon..." right after server startup. The timer will now correctly display the full interval countdown.
  • Corrected the tax countdown timer display in the GUI to use real-world time instead of Minecraft ticks, keeping it consistent with the actual tax invoice generation logic.
  • Completely overhauled TaskManager scheduling for Income and Tax events to use true real-time epoch tracking instead of simple Bukkit task delays. This guarantees timers are 100% accurate, synchronizes flawlessly with MyApartmentsGUI, and ensures backdated tasks are caught up properly if the server is offline.
  • Removed redundant tax evaluation executions in processDailyUpdates to prevent double-charging invoices.
  • MyApartmentsGUI "Next Income In" lore is now tax-status aware. It will display a clear warning (OVERDUE, INACTIVE, REPOSSESSION) instead of a countdown if income is blocked due to unpaid taxes, and will hide unrelated research buffs.
  • Implemented the missing "High Speed Internet" shop buff functionality. It now correctly amplifies income amounts sequentially to mathematically compensate for tick interval reductions, maintaining server performance without breaking the global timer logic.
  • Fixed an issue where buff_values and costs for Shop Items (including "High Speed Internet") were hardcoded in the plugin enum. They are now fully dynamic and read in real-time from shop.yml.
  • Fixed a similar issue in the Research system where effect-per-tier, effect-per-tier-fee, and effect-per-tier-commission math logic was statically hardcoded. Value multipliers are now strictly parsed directly from research.yml to allow endless configurability.
⚙
Configuration
  • New section achievements.effects in achievements.yml:
    • sound - toggle, sound name, volume, pitch
    • title - toggle, header, subheader, fade-in, stay, fade-out (all in ticks)
    • action-bar - toggle, format string (supports {achievement} placeholder)
    • bossbar - toggle, title, color, style, duration-ticks
  • New field tax-percentage in config.yml under each apartment level setting.
  • All effects support enabled: false to individually disable them.
  • Plugin logs a warning if an invalid sound or bossbar color/style name is configured.
↑
Technical
  • AchievementManager now reads and caches all effect config fields on load/reload.
  • Models updated to use the dynamic taxPercentage value from LevelConfig instead of the hardcoded multiplier.

New Features
  • Added Income Capacity mechanism so apartment income cannot accumulate infinitely; each apartment level has a maximum pendingIncome accumulation limit (Level 1 = 10,000, Level 2 = 20,000, etc., increasing by 10,000 per level).
  • Automatic income generation is stopped when pendingIncome reaches full capacity; new income will not be added until the player claims it.
  • If income exceeds remaining capacity, the value is automatically clamped so pendingIncome never exceeds the level limit.
  • Online players receive a chat notification [Capacity Full] when their apartment's income vault is full, along with level capacity info.
  • Added income-capacity field to each level in the apartment-levels block in config.yml for easy per-level capacity configuration.
  • Added a duration system to the apartment upgrade process; upgrades are no longer instant but require completion time.
  • Added upgrade-duration config attribute (in ticks) in config.yml for each level in apartment-levels (0 = instant).
  • Added a special confirmation GUI (UpgradeConfirmGUI) that appears when a player presses the Upgrade button. This GUI presents Price, Remaining Duration, and Estimated New Income info before final confirmation.
  • Changed the upgrade button in the ApartmentDetailsGUI to a remaining time indicator (Upgrading...) when the apartment is in the level renovation phase.
  • Added an automatic checker in TaskManager.java that processes apartments that finish upgrading every second (20 ticks), accompanied by Title notification and Level-Up sound effects for online players.
  • Added Quick Sell mechanism: apartment owners can directly sell their apartment back to the government using the existing sell-percentage configuration in config.yml (default 70% of purchase price).
  • Added Market Sell mechanism: apartment owners can post listings in the apartment browser so other players can buy the apartment with an ownership transfer mechanism. The listing price matches the original apartment price, and the payment goes directly to the seller.
  • In ApartmentBrowserGUI, market listings display seller information, apartment rating, apartment level, income range, and a 📢 Market Listing label to distinguish them from government properties.
  • In ApartmentBrowserGUI, apartments sold/created by admins (without owners) now display a 🏛 Government Property label and Owner: Government on the item lore.
  • Changed the /ac sell <apartment_id> command to /ac sell quick <apartment_id> for quick sell and /ac sell market <apartment_id> for market sell, and /ac sell cancel <apartment_id> to cancel a market listing.
  • Added /ac marketbuy <apartment_id> command for buying an apartment from a market listing (with confirmation mechanism).
  • In ApartmentDetailsGUI, the Sell button is replaced with two separate buttons: ⚡ Quick Sell (slot 24) and 📢 Market Sell (slot 25). If the apartment is already listed on the market, the Market Sell button changes to Listed on Market with an option to click to cancel the listing.
  • In ApartmentDetailsGUI, added a 🛒 Buy from Market button for non-owners when the apartment is listed on the market, displaying seller info and market price.
  • Added Vault Capacity (income-capacity) info to the info item lore in ApartmentDetailsGUI (apartment detail menu).
  • Improved the Pending Income display format in MyApartmentsGUI to also show the maximum limit / Vault Capacity (example: $1,000 / $10,000).
  • Changed the Market Sell mechanism in ApartmentDetailsGUI. Now when a player clicks the Market Sell button, the GUI will close and the game will ask the player to input a custom price (desired selling price) via chat, accompanied by price format validation. The player can type 'cancel' to cancel the listing process.
  • Added a new research type capacity_expansion (Vault Expansion) to the Research System, which provides a +5% Income Capacity bonus per tier. This research recipe is made expensive with the longest research time duration.
  • Added research bonus calculation display to the Next Income In item lore in MyApartmentsGUI, displaying the percentage bonus of capital_growth (CG) and revenue_acceleration (RA) for players who have completed the research (example: (+15% CG, -10% RA)).
  • Added admin command /ac admin set research <factory_id> <research_id> <tier> to manually increase research level, useful for testing or events.
  • Added total research level amount info (Research Lv) owned by the seller to the lore item of apartments sold on the market (ApartmentBrowserGUI).
  • Added tax_efficiency (Tax Efficiency) buff effect display to the tax value lore item in TaxManagementGUI, displaying the tax reduction percentage from research (example: -15% (Research)) on the Tax Overview, Apartment Tax Item, and Invoice Item.
  • Added full tab completion for the /ac admin set research <factory_id> <research_id> <tier> command, supporting auto-complete for apartment IDs, research types (ResearchType), and maximum tier limits.
  • Added a complete Achievement system with 5 achievement types: Income Millionaire (accumulate $1,000,000 income), Tax Contributor (accumulate $1,000,000 tax), Sales Tycoon (accumulate $1,000,000 sales), Research Master (complete 1 research to max tier), and Max Level Owner (own 1 max level apartment).
  • Added a custom AchievementGUI showing real-time progress bars, completion percentages, and reward info for each achievement.
  • Connected the Achievement GUI to the Main Menu GUI with a Nether Star button in slot 30, showing a summary of the player's achievement progress.
  • Added real-time achievement tracking on related events: income claim, tax payment (manual & auto-pay), apartment sales (quick & market sell), research completion, and apartment upgrade to max level.
  • Added achievement unlocked notification with chat message, title screen, and sound effect (UI_TOAST_CHALLENGE_COMPLETE) when players complete an achievement.
  • Added a global broadcast system that announces achievement completions to all online players.
  • Added automatic money rewards when an achievement is completed (default: $50,000 per achievement, configurable).

New Feature — Research System
  • Introduced a new Research System that grants permanent player-level buffs through tier-based research (Tier I–V).
  • Created research.yml configuration file with configurable cost (exponential scaling), duration (exponential scaling), and effect values per research type.
  • Created data/research_data.yml for persistent storage of player research progress and active research state.
  • Implemented 5 research types:
    • Revenue Acceleration — Reduces income generation interval by 5% per tier (applied as a proportional income boost).
    • Capital Growth Strategy — Increases income generation amount by 5% per tier.
    • Tax Efficiency Strategy — Reduces final tax amount by 5% per tier (stacks with shop tax reduction).
    • Expansion Plan — Increases apartment ownership limit by +1 per tier.
    • Auction Efficiency — Reduces the auction creation fee by 5% per tier and lowers the commission rate by 1% per tier.
  • Research costs scale exponentially: cost = base × multiplier^(tier-1). Default base values range from $50,000 to $150,000.
  • Research durations scale exponentially: duration = base × multiplier^(tier-1). Default base values range from 1h to 3h.
  • Players can only run one research at a time; completion is checked every second.
  • On research completion, the player receives a chat notification, title message, and sound effect.
  • New files added:
    • ResearchType.java — Enum defining the 5 research types with config keys, display names, descriptions, icons, and max tier.
    • PlayerResearchData.java — Model class storing per-player completed tiers and active research state.
    • ResearchManager.java — Core manager handling config loading, cost/duration calculations, research start/complete/cancel, buff getters, tick processing, and YAML persistence.
    • ResearchGUI.java — Custom 54-slot GUI displaying all 5 researches with tier progress bars, buff descriptions, cost/duration info, and active research tracking with countdown.
    • research.yml — Configuration file for the research system.

    • AuctionManager.java — Applied Auction Efficiency buff to reduce auction fee in createAuction() and commission rate in processAuctionEnd().
    • CommandHandler.java & ApartmentCommandService.java — Applied Expansion Plan buff to the max apartment ownership limit in the buy command.
    • messages.yml — Added research.* message entries: started, completed, already_active, max_tier, insufficient_funds, disabled, no_permission.
    • plugin.yml — Added apartmentcore.research permission (default: true).
